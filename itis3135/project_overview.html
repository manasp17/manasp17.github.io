Project Overview

Application and Purpose:

The Basal Metabolic Rate Calculator is a web application that allows users to calculate their BMR based on personal details. This BMR value will then be used to determine the number of calories needed to maintain, gain, or lose weight. The purpose of the site is to provide an easy-to-use tool that can help individuals understand their caloric needs based on their activity level and body data.

Intended Users:

The intended users are individuals who are interested in understanding their calorie needs for weight management. These users could include individuals interested in fitness, weight loss, or those trying to gain muscle mass.

Overview of the Content:

The website will consist of a homepage (with an introduction and the BMR calculator), informational pages about BMR, and results pages that provide insights based on the user’s inputs. Users will also have access to a FAQ section about nutrition and BMR.

Site Map/Wireframe

The site map has pages that make up the site and indicate how people will navigate through them, below is the structure of the website.

  • Home Page: Introduce the tool and allow users to input data for BMR calculation.
  • Calculator Page: Input data for BMR calculation like age, gender, weight, and height.
  • Results Page: Display the calculated BMR and recommended daily calorie intake for maintenance, weight gain, or weight loss.
  • FAQs Page: Answer common questions about BMR and Nutrition.
  • Contact Page: Provide a way for users to get in touch for inquiries.

Page Design

1. Home Page

Purpose: Introduction to the BMR Calculator and general website overview.

Audience/Users: Visitors who are learning about BMR and want to use the tool.

Content: Brief introduction and a link to the calculator page.

Data Fields: No data entry on this page.

Links: Link to the BMR calculator page.

Buttons/Actions: Button to navigate to the calculator page.

2. Calculator Page

Purpose: Allow users to input personal details (age, height, weight, gender) to calculate BMR.

Audience/Users: Anyone interested in calculating their BMR.

Content: Input fields for age, height, weight, and gender.

Data Fields: Yes, these fields will need validation (non-empty and valid number entries).

Links: Link to results page after form submission.

Buttons/Actions: Submit button to send input data for BMR calculation.

3. Results Page

Purpose: Display the calculated BMR and caloric needs based on activity level.

Audience/Users: Users who have completed the BMR calculator.

Content: Display the BMR, maintenance calories, and options for gaining or losing weight.

Data Fields: No data entry here, just displaying calculated results.

Links: Return to the calculator or home page.

Buttons/Actions: Reset button to return to the calculator page.

4. FAQs Page

Purpose: Provide answers to common questions regarding BMR and its importance in weight management.

Audience/Users: General users looking for more information.

Content: List of frequently asked questions and answers.

Data Fields: No data entry here.

Links: Return to other pages on the site (home, calculator, results).

Buttons/Actions: Link to various parts of the website.

5. Contact Page

Purpose: Allow users to contact the website’s owner for any inquiries or feedback.

Audience/Users: Users with specific inquiries or feedback.

Content: Contact form with fields for name, email, and message.

Data Fields: Yes, all fields must be validated.

Links: None.

Buttons/Actions: Submit button for sending messages.

Dynamic Functionality

  • Forms: The primary dynamic feature will be the BMR calculation form where users input data, they submit it and JS will process the BMR calculation.
  • Calculator: Once the user submits the data, JS will calculate it and display the results on the calculator page.
  • AJAX: Will be used to update the results page dynamically without refreshing the page, will improve user experience.
